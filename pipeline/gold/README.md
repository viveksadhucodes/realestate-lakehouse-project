# Gold Layer Analytics Guide

The Gold layer is the decision layer of this lakehouse project. It transforms clean Silver data into KPI-ready outputs for dashboards, business reporting, and stakeholder decisions.

## Visual Snapshot

| Layer | Purpose | Typical Output |
|---|---|---|
| Bronze | Raw ingestion with lineage | Raw Delta tables |
| Silver | Cleaning, standardization, entity integration | Trusted fact and dimensions |
| Gold | Business analytics and KPI modeling | Aggregated KPI tables/views |

In this project, Gold logic is implemented in analytics.py and runs on top of the Silver fact table.

## What Gold Solves

Gold functions answer high-value business questions such as:

- How much revenue is generated by city, property type, and category?
- Which agents and channels are best performing?
- How fast is inventory selling?
- What buyer patterns are emerging by income segment and purchase behavior?

## Source and Expected Input

- Logic file: pipeline/gold/analytics.py
- Expected input DataFrame: Silver fact data with columns such as:
  - transaction_id, customer_id, property_id, agent_id
  - city, property_type, property_category
  - deal_date, year_month, deal_price, commission_amount
  - income_band, listing_channel, days_on_market, sold_flag
  - city_variation_flag, agent_city

## KPI Catalog

### Core KPIs

1. revenue_by_city(df)
- Total revenue by city
- Output: city, total_revenue

2. monthly_sales(df)
- Sales volume trend over year_month
- Output: year_month, total_sales

3. top_agents(df)
- Agent ranking by total commission
- Output: agent_id, total_commission, rank

4. top_expensive_properties(df)
- Top 10 transactions by deal price
- Output: top priced rows from fact

5. avg_price_by_property_type(df)
- Average deal price by property type
- Output: property_type, avg_price

6. customer_purchase_frequency(df)
- Frequency of purchases by customer
- Output: customer_id, purchase_count

7. property_demand(df)
- Demand and selling profile by city
- Output: city, total_transactions, avg_days_on_market, sold_properties

8. buyer_type(df)
- Classifies customer as New or Repeat
- Output: customer_id, purchase_count, buyer_type

9. running_revenue(df)
- Cumulative revenue over deal timeline
- Output: original columns + running_revenue

10. agent_ranking(df)
- Rank agents within each city by generated revenue
- Output: agent_id, agent_city, revenue, rank

### Advanced KPIs

11. revenue_by_property_category(df)
- Revenue contribution by property category
- Output: property_category, total_revenue

12. sales_by_income_band(df)
- Sales distribution by income segment
- Output: income_band, total_sales

13. commission_efficiency(df)
- Average commission ratio per agent
- Output: agent_id, avg_commission_ratio

14. listing_conversion_rate(df)
- Conversion effectiveness of listing channels
- Output: listing_channel, total_listings, sold_count, conversion_rate

15. city_variation_impact(df)
- Impact analysis where city and property city differ
- Output: city_variation_flag, city, total_transactions, avg_price

16. market_speed_analysis(df)
- Market speed by property type
- Output: property_type, avg_days, transactions

### Behavioral KPIs

17. fastest_selling_category(df)
- Fastest moving property categories
- Output: property_category, avg_days_on_market, total_sales

18. high_value_buyers(df)
- Average deal value by income band
- Output: income_band, avg_purchase_value, total_transactions

19. premium_buyers(df)
- Premium segment purchasing behavior (high category)
- Output: income_band, premium_purchases, avg_price

## Design Strengths

- Modular function design for easy testing and reuse
- Consistent Spark DataFrame transformations
- Window functions for ranking and cumulative analysis
- KPI outputs are analysis-friendly and dashboard-ready
- Compatible with notebook execution and batch jobs

## Suggested Execution Pattern

1. Load Silver fact table into a DataFrame.
2. Import analytics functions.
3. Run only the KPIs needed by the business use case.
4. Persist outputs as Gold tables or views.
5. Feed BI tools from these curated outputs.

## Example Usage

```python
from pipeline.gold.analytics import (
    revenue_by_city,
    monthly_sales,
    top_agents,
    listing_conversion_rate
)

fact_df = spark.table("silver.silver_real_estate_fact")

kpi_revenue_city = revenue_by_city(fact_df)
kpi_monthly = monthly_sales(fact_df)
kpi_top_agents = top_agents(fact_df)
kpi_channel_conversion = listing_conversion_rate(fact_df)

kpi_revenue_city.write.mode("overwrite").saveAsTable("gold.kpi_revenue_by_city")
kpi_monthly.write.mode("overwrite").saveAsTable("gold.kpi_monthly_sales")
kpi_top_agents.write.mode("overwrite").saveAsTable("gold.kpi_top_agents")
kpi_channel_conversion.write.mode("overwrite").saveAsTable("gold.kpi_listing_conversion")
```

## Data Quality Notes

Gold assumes Silver has already validated:

- primary key deduplication at transaction grain
- commission integrity checks
- date sanity checks
- key join consistency

If KPI values look unusual, validate Silver first before changing Gold logic.
